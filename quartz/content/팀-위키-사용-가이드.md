# 팀 위키 사용 가이드

팀 위키는 팀의 지식 관리 및 문서화 효율성 향상을 목표로 합니다. 본 문서는 팀원들이 위키에 효과적으로 기여하고 활용하는 방법을 안내합니다.

## 콘텐츠 기여 방법
팀 위키에 콘텐츠를 추가하는 세 가지 방법이 있습니다. 가장 편리한 방법을 선택하여 사용하세요.

### 1. GitHub 웹 인터페이스
가장 간단한 방법으로, 웹 브라우저만 있으면 언제든지 기여할 수 있습니다.

1.  `MINO-WIKI-PRIVATE` 저장소로 이동합니다.
2.  `raw/` 폴더로 들어갑니다.
3.  우측 상단의 `Add file` 버튼을 클릭한 후 `Create new file`을 선택합니다.
4.  파일명은 `YYYY-MM-DD-제목.md` 형식으로 지정하는 것을 권장합니다. (예: `2026-04-20-배포회의.md`). 정확한 파일명 규칙은 AI가 자동으로 정리하므로, 필수는 아닙니다.
5.  내용을 작성한 후 하단의 `Commit changes` 버튼을 클릭합니다.
6.  약 2~3분 후, 정리된 내용이 `wiki/` 폴더에 생성되고 공개 위키에 반영됩니다.

### 2. Obsidian (옵시디언)
노트 작성을 자주 하는 팀원에게 가장 추천하는 방법입니다. Obsidian의 다양한 기능을 활용하여 효율적인 문서 작성이 가능합니다.

1.  `MINO-WIKI-PRIVATE` 저장소를 로컬에 `clone`합니다.
2.  Obsidian을 실행하고 클론한 저장소 폴더를 `Vault`로 엽니다.
3.  최초 1회만 `Community Plugins`를 활성화하고 `Obsidian Git` 플러그인을 설치합니다. (`.obsidian` 설정은 이미 완료되어 있습니다.)
4.  설치 후 왼쪽 사이드바에 나타나는 `Git` 패널을 통해 `Commit` 및 `Push` 작업을 수행합니다. 별도의 터미널 작업이 필요 없습니다.
5.  `Cmd + G` 단축키를 사용하여 그래프 뷰를 통해 지식 간의 연결 관계를 시각적으로 확인할 수 있습니다.

### 3. 터미널 (Git)
개발자에게 효율적이고 익숙한 방법입니다.

1.  로컬 `MINO-WIKI-PRIVATE` 저장소로 이동합니다.
2.  `git pull` 명령으로 최신 변경사항을 가져옵니다.
3.  `raw/` 폴더 안에 새로운 `.md` 파일을 생성하고 내용을 작성합니다.
4.  `git add`, `git commit`, `git push` 명령을 순차적으로 실행하여 변경사항을 반영합니다.

## 권장 콘텐츠 종류
다음과 같은 종류의 콘텐츠가 위키에 잘 정리되며, 팀의 지식 축적에 큰 도움이 됩니다.

### 회의록
팀 회의에서 결정된 사항이나 논의 내용을 기록합니다. 불릿 포인트 형식으로 작성해도 AI가 섹션을 나누어 구조화합니다. (예: "A가 ~하기로 함, B는 ~를 맡기로 함")

### 트러블슈팅 로그
발생한 에러와 해결 과정을 상세히 기록합니다. 현재의 해결책을 기억하기 위함뿐만 아니라, 향후 다른 팀원이 유사한 문제를 겪을 때 유용한 참고 자료가 됩니다. (예: "Xcode 16 pod install 안됨 해결")

### 기획 아이디어
아이디어를 러프하게 제안하고 장단점을 간략히 기술해도 AI가 이를 구조화된 문서로 변환합니다.

### 팀 용어집
팀 내부에서 사용되는 약어, 전문 용어 등을 정의하여 신규 팀원의 온보딩을 돕습니다.

### 온보딩 문서
신규 팀원을 위한 개발 환경 설정, 주요 저장소 목록 등 온보딩에 필요한 정보를 정리합니다.

## 일반 사용자(외부인)를 위한 공개 위키
팀 위키의 내용은 정제되어 외부에 공개될 수 있습니다.

*   **공개 사이트 주소:** `hooni0918.github.io/MINO-WIKI-PUBLIC`
*   **접근성:** 누구나 접근하여 내용을 열람할 수 있습니다.
*   **구현 기술:** Quartz 기반의 정적 페이지로, 깔끔하고 기능적인 인터페이스를 제공합니다.

### 공개 사이트 주요 기능
*   **파일 탐색기:** 좌측 사이드바에서 폴더별 파일 목록을 탐색할 수 있습니다.
*   **백링크:** 현재 페이지를 참조하는 다른 페이지 목록이 우측에 표시됩니다.
*   **검색:** 상단 검색창을 통해 위키 전체 내용을 빠르게 검색할 수 있습니다. (단축키: `Cmd + K` 또는 `Ctrl + K`)
*   **다크 모드:** 다크 모드 토글을 지원합니다.
*   **그래프 뷰:** 모든 페이지 간의 연결 관계를 시각적으로 보여줍니다.
*   **목차 및 위키링크:** 각 페이지 상단에 자동으로 목차가 생성되며, 헤딩 클릭 시 해당 섹션으로 이동합니다. `[[파일명]]` 형식의 위키링크 클릭 시 해당 페이지로 이동하며, 마우스 오버 시 미리보기가 표시됩니다.

### Private/Public 분리
`MINO-WIKI-PRIVATE` 저장소의 `raw/` 폴더에 있는 원본 파일은 외부에 절대 노출되지 않습니다. 오직 AI에 의해 정제되고 구조화된 `wiki/` 폴더의 내용만 `MINO-WIKI-PUBLIC` 저장소를 통해 공개됩니다. 따라서 러프한 메모나 중간 단계의 아이디어는 안전하게 비공개로 유지됩니다.

## 자동 정리 프로세스 (기여자 관점)
`raw/` 폴더에 작성된 내용은 [[팀-위키-자동화-시스템-구성]]에 따라 AI를 통해 다음과 같이 자동으로 정제됩니다.

*   **어조 변환:** 반말체 문장이 존댓말로 변경됩니다.
*   **구조화:** 두서없는 의식의 흐름 글이 섹션으로 나뉘어 구조화됩니다.
*   **마크다운 서식 적용:** 목록, 헤딩, 코드 블록 등이 적절한 위치에 자동으로 적용됩니다.
*   **위키링크 자동 삽입:** 내용에서 언급된 관련 개념에 `[[위키링크]]`가 자동으로 생성됩니다.
*   **콘텐츠 병합:** 동일 주제의 기존 페이지가 있는 경우, 새로운 내용이 기존 페이지에 병합됩니다.
*   **인덱스 페이지 재분류:** 카테고리별 인덱스 페이지가 자동으로 재분류됩니다.

### 수정 및 보완
AI의 자동 정리 결과가 기대와 다른 경우, `raw/` 원본 파일을 다시 수정하여 업로드하거나, `wiki/` 폴더의 파일을 직접 수정할 수 있습니다.

## 기술 스택
팀 위키 자동화 시스템은 다음의 기술 스택을 활용하여 무료로 운영됩니다.

*   **[[Gemini Flash API]] 2.5:** Google의 AI 모델로, 문서 정리를 담당합니다. 무료 티어(분당 15회, 일 1500회)는 팀 규모에 충분합니다.
*   **GitHub Actions:** CI/CD 자동화를 담당합니다. 무료로 제공됩니다.
*   **Quartz v4:** 마크다운 파일을 정적 웹사이트로 변환하는 오픈소스 도구입니다.
*   **Obsidian:** 마크다운 에디터 (선택 사항). 개인 용도로 무료 사용이 가능합니다.
*   **GitHub Pages:** 정적 웹사이트 무료 호스팅을 제공합니다.

## 주의사항

*   **보안:** `raw/` 폴더에 API 키, 비밀번호, 개인 정보(주민등록번호, 전화번호, 주소 등)와 같은 민감한 정보를 직접 작성하지 마세요. 비공개 저장소라 할지라도 Gemini로 전송되는 과정에서 잠재적 위험이 있을 수 있습니다.
*   **파일 삭제:** `raw/` 폴더에서 파일을 삭제하더라도 `wiki/` 폴더의 공개된 파일은 자동으로 삭제되지 않습니다. 공개 위키에서 파일을 제거하려면 `wiki/` 폴더에서 해당 파일을 수동으로 삭제해야 합니다.
*   **파일 수정:** `raw/` 파일을 수정하면 파일의 해시값이 변경되어 AI가 해당 파일을 다시 처리합니다. (내용이 동일한 파일을 다시 업로드하는 경우는 건너뛰며, 이 경우에는 다시 처리되지 않습니다.)

## 유용한 팁

*   **Obsidian 그래프 뷰:** Obsidian을 사용하면 지식 간의 연결 관계를 시각적으로 파악할 수 있습니다.
*   **공개 사이트 검색:** 공개 사이트에서 `Cmd + K` (또는 `Ctrl + K`) 단축키를 눌러 전체 검색창을 빠르게 활성화할 수 있습니다.
*   **태그:** 본문 내에 `#태그이름` 형식으로 작성하면 Quartz가 자동으로 태그를 인식하여 분류에 활용합니다.
*   **이미지 첨부:** 이미지는 `raw/attachments/` 폴더에 저장하고 마크다운 문법으로 참조하여 첨부할 수 있습니다.
